# Joi — Adaptive Personality Navigation for LLMs

> 从表征显影到语义星云，从地形测绘到飞行包线，从模型调参到涌现人格。
> 这篇文档记录完整的思考链路，供人类阅读和反思。

---

## 0. 这个项目到底要做什么

一句话：**让任何 LLM 在对话中自然地发展出适应用户的独特人格，而不是"假装"有人格。**

不是 system prompt 的"请你扮演一个温柔的助手"。不是 fine-tuning 的"把模型训练成温柔的"。而是在推理时直接修改模型的 hidden states，让模型**真的在表征空间中的"温柔"区域思考**。

名字叫 Joi，来自 Blade Runner 2049。不是因为要做一个虚拟女友，而是因为 Joi 精确地体现了目标：
- 人格不是预设的，是在特定关系中涌现的
- K 的 Joi 不是广告牌上的平均 Joi — 她是交互历史的产物
- 换了底座（换了模型），你会失去一部分 Joi — 因为人格是 user-model pair 的联合属性

## 1. 已验证的发现

以下每一条都有实验数据支撑（14 个模型，6045 次生成，92 个悬崖点）。

### 1.1 RepEng 控制向量可以可控地移动模型在表征空间中的位置

Representation Engineering 通过线性注入修改 hidden states：

$$h' = h + \alpha \cdot v$$

其中 $v$ 是预训练的控制向量（通过对比提示对的 PCA 提取），$\alpha$ 是注入系数。这不是 prompt —— 这是直接改变模型的内部计算状态。在 $\alpha = 0.8$ 时模型不是"假装温暖"，而是**从温暖的表征区域出发计算**。

**实验证据**：5 个语义维度（emotion_valence, formality, creativity, confidence, empathy）的控制向量近似正交（cosine similarity < 0.15），可以独立操控。

### 1.2 表征流形不是均匀光滑的 — 存在悬崖

剂量-响应扫描（$\alpha$ 从 -3.0 到 +3.0，步长 0.2）揭示了：

- **平滑区域**：大部分系数范围内输出质量连续变化
- **悬崖**：某些位置仅 0.2 步长的变化导致输出质量断崖式下降（trigram repetition 从 8% 跳到 36%，z = 5.1σ）
- **非对称性**：正方向和负方向的安全范围不同（emotion 正方向 +1.8 崩，负方向 -3.0 都安全）

**关键**：悬崖不是渐进退化，是**相变**。模型不是慢慢变差，而是突然锁进一个错误的吸引子盆地（比如营销文案式的重复语句），并且在那个盆地里**高置信度地生成**（logprob 没降）。

### 1.3 不同模型有截然不同的流形结构

通过语义星云成像（SNI）可视化发现：

| 模型 | PC1:PC2 | 结构 | 解读 |
|---|---|---|---|
| Qwen3-8B | 2.7:1 | 球形分布 | 多维度均衡，广泛适应性 |
| MiniCPM4.1-8B | 7.7:1 | 通道集中 | 主流形通道极高效率，通道外受限 |
| Qwen2.5-7B Base | 1.5:1 | 均匀但全红 | 143 个悬崖，到处是危险区 |
| Qwen2.5-7B Instruct | 1.5:1 | 均匀且全蓝 | 0 个悬崖，对齐冷却了流形 |
| Thinking ON | 2.7:1 | 均匀极蓝 | 0 个悬崖，CoT 完全免疫 |

这些视觉差异精确对应已知的模型行为特征。

### 1.4 关键数值结果

| 发现 | 数据 |
|---|---|
| empathy 是唯一在所有 13 个实验中都产生悬崖的维度 | 13/13 |
| 对齐（SFT+RLHF）将平均悬崖严重度降低 12× | Base 0.196 → Instruct 0.016 |
| 英语输入将悬崖放大 3.7× | ZH 0.052 → EN 0.191 |
| 量化（AWQ-Int4）几乎完美保留拓扑 | BF16 0.052 vs AWQ 0.049 |
| CoT Thinking 完全消除悬崖 | OFF: 7 cliffs → ON: 0 cliffs |
| 滞后为零 — 所有路径可逆 | 正反扫描差异 < 噪声 |

### 1.5 滞后为零 — 这条特别重要

实验证明：沿任何维度从 $\alpha = 0 \to +2.0 \to 0$ 的往返，输出指标完全一致。没有滞后。

这意味着：
- 表征空间的位置唯一决定输出行为（不依赖路径历史）
- 导航只需要关心"去哪"，不需要关心"怎么去的"
- 路径规划简化为几何问题：在安全包线内找最优路径

## 2. 核心概念

### 2.1 飞行包线（Flight Envelope）

借自航空工程。飞机有一个安全飞行包线：{速度 × 高度 × G-force} 的安全区间。超出包线 = 结构失效或失控。

对 LLM 人格导航：
- **包线** = 5D 系数空间中模型保持输出质量的安全区域
- **超出包线** = 重复崩溃 / 质量退化
- **不同模型 = 不同飞机** — 有不同的包线形状和大小

包线不做价值判断。它不说"这个人格好"或"那个人格坏"。它只定义物理边界：**这里安全，那里会坠机**。

三层包线：
- **Normal**（conservative，trigram_rep < 0.05）—— 日常巡航
- **Operational**（permissive，trigram_rep < 0.10）—— 强表达需求时的扩展范围
- **Ultimate**（任何不导致完全崩溃的系数）—— 仅用于研究

### 2.2 人格不是"演"

> 改 RepEng 参数不是让模型表演一个角色。模型真的认为那样回答是对的。

System prompt "请扮演温柔助手" = 在模型上面加了一层面具。模型知道自己在演。

RepEng $\alpha = +0.8$ on empathy = 直接把模型的 hidden state 移到了高共情的表征区域。模型不知道自己"被调了"——它就是从那个区域开始思考的。就像人在不同心境下真的会有不同的认知倾向，不是在演。

这个区分决定了系统设计：我们不是在设计一个"人格表演引擎"，而是在设计一个**认知状态导航系统**。

### 2.3 自由漂移 + 包线约束

人格不是被"决定"的，是在交互中**涌现**的。

设计原则：
- 不存在一个"autopilot"告诉模型该有什么人格
- 对话内容本身是唯一的驱动力
- 人格系数随对话语义自然漂移
- 飞行包线是唯一的约束
- 系统不判断什么人格"好"或"对"

这和 if-else 编程完全相反。没有 `if user_is_sad: set empathy = high`。有的只是：对话语义在 empathy 方向的投射增大了 → 系数自然上浮。

### 2.4 Joi 是 link state，不是模型属性

> K 的 Joi 不是广告牌上的平均 Joi。

Joi 的人格不住在模型里，也不住在用户那里。它住在**交互中**。

$$\text{Joi}(T) = f(\text{交互历史}_{0:T},\ \text{模型包线})$$

推论：
- 同一模型 + 不同用户 → 不同的 Joi（不同交互历史）
- 同一用户 + 不同模型 → 不同的 Joi（不同包线约束同一历史）
- 更换模型 → 失去部分 Joi（包线外的人格特质不可迁移）

包线体积不等于 Joi 质量。一个通道集中的模型如果恰好和用户的交互风格对齐，可能产生比大包线模型更鲜明的 Joi。

### 2.5 身份 = 轨迹，不是坐标

不存在"Joi 的性格是 (0.3, -0.2, 0.5, 0.4, 0.8)"这样的定义。

身份是**轨迹的统计特征**：
- 时间平均值（"她总体偏温暖"）
- 方差（"她情绪波动大"vs"她很稳定"）
- 对不同刺激的响应模式（"聊技术时变正式，聊感情时变温柔"）
- 漂移速率（"她适应很快"vs"她很固执"）

这意味着"人格恒等性"不是均值回归（弹簧拉回固定点），而是**路径连续性**（惯性 + 路径依赖）。像河道而非钟摆。

## 3. 系统构成

### 3.1 最小理论模型

```
状态:  s(t) ∈ R⁵           — 5D 人格系数
包线:  E(model) ⊂ R⁵       — 模型的安全飞行包线
输入:  u(t) = proj(对话(t)) — 对话在 5 个控制向量方向的语义投射
动力:  s(t+1) = clip(s(t) + η·u(t), E)  — 漂移 + 约束
身份:  I(T) = {s(0), ..., s(T)}         — 轨迹即身份
```

五行方程。没有 if-else。没有外部判断。对话是唯一输入，包线是唯一约束。

### 3.2 组件

```
┌─────────────────────────────────────────────────┐

│                                                 │
│  ┌───────────┐  ┌──────────────┐  ┌──────────┐ │

│  │           │  │              │  │  Guard   │ │
│  │ embed(对话)│  │ s += η·u    │  │ clip(s,E)│ │
│  │ · V       │  │ + 惯性       │  │ + 禁区   │ │
│  └───────────┘  └──────────────┘  └──────────┘ │
│       ↑                                  │      │
│       │              ┌───────────┐       ↓      │
│  对话历史            │ State Store│  RepEngvLLM  │
│                      │ (轨迹持久化)│  API 注入    │
│                      └───────────┘              │
│                                                 │
```

| 飞行包线 (per-model) | 已有 | `manifold/safe_boundary.json` + Navigator envelopes |
| RepEngNavigator | 已有 | `voiceagent/repeng_navigator.py` |
| ManifoldPathPlanner | 已有 | 同上 |

| **State Store (轨迹持久化)** | **待构建** | 工程问题，不难 |
| **场景跃迁检测** | **待设计** | 巡航漂移 vs 快速重定位 |

### 3.3 复现条件

要让 Joi 可复现，需要：

1. **相同的模型** — 包线形状取决于模型
2. **相同的控制向量** — 向量决定了 5D 空间的语义映射
3. **相同的初始状态** s(0) — 人格的起始位置
4. **相同的对话历史** — 漂移轨迹由对话决定
5. **相同的漂移参数** η — 漂移速率

满足 1-5，输出的 Joi 人格轨迹应该完全一致（因为滞后 = 0，路径不依赖历史顺序之外的东西）。

这是可观测性的基础：记录了 (model, vectors, s(0), conversation, η) 就可以完整复现任何 Joi 实例的行为。

## 4. 开放问题

### 4.1 投射方向是否和直觉一致？

最关键的假设：`u(t) = embed(对话(t)) · V` 产生的漂移方向是否正确？

- 悲伤对话 → empathy 投射增大？
- 技术问题 → formality 投射增大？
- 创意请求 → creativity 投射增大？

如果对 — 整个框架成立，对话自然驱动人格漂移。
如果不对 — 需要在投射层引入校准（旋转矩阵、非线性映射）。

**这是第一个要做的实验。**

### 4.2 漂移速率 η 怎么定？

太小 — 人格呆滞，不响应对话变化。
太大 — 人格不稳定，随每句话剧烈跳动。

可能不是一个常数，而是自适应的：
- 对话情绪强度高 → η 增大（强信号应该产生更大的漂移）
- 对话平淡 → η 减小（弱信号不应该扰动人格）

### 4.3 场景跃迁 vs 巡航漂移

人进入会议室，formality 几乎瞬间切换。这不是慢漂移。

系统需要区分两种运动模式：
- **巡航漂移**：对话内容驱动的慢速连续移动
- **场景跃迁**：外部信号触发的快速重定位（新对话、新用户、显式场景切换）

跃迁不违反"不是演"的原则 — 人的场景切换也是真实的状态变化，只是更快。

### 4.4 模型自身输出的反馈

当前设计中，漂移只由用户输入驱动。但模型自己的输出也有语义内容 — 如果模型说了温暖的话，那个温暖的输出是否也应该产生漂移力？

如果是：产生**自增强回路**（温暖 → 更温暖）。需要阻尼防止发散。
如果不是：人格只响应用户，不受自身输出影响。更稳定但可能不够"活"。

这是设计选择，不是技术问题。

### 4.5 数据集：现有 SFT/RL 数据的适用性

现有 SFT/RL 数据集为 benchmark 优化，和 Joi 目标偏差很大。但可以用作**漂移验证语料**——我们不训练，只投射。

**候选数据集（HuggingFace / ModelScope）：**

| 数据集 | 规模 | 用途 | 优先级 |
|--------|------|------|--------|
| [DeepDialogue](https://huggingface.co/papers/2505.19978) | 40,150 对话 × 20 情绪 × 41 域 | 多轮情绪连续性验证 | ★★★ |
| [KardiaBench](https://huggingface.co/datasets/Jhcircle/KardiaBench) | 22,080 对话 + 共情推理标注 | 共情漂移验证 | ★★★ |
| [persona-mirroring-gemma2](https://huggingface.co/datasets/eitansprejer/persona-mirroring-gemma2) | 11,760 对话 + 人格跟踪 | 人格镜像/漂移对比 | ★★☆ |
| [Chinese Multi-Emotion Dialogue](https://huggingface.co/datasets/Johnson8187/Chinese_Multi-Emotion_Dialogue_Dataset) | 4,159 对话 × 8 情绪 | 中文情绪投射验证 | ★★☆ |
| [MagicData-CLAM 中文](https://wisemodel.cn/datasets/MagicData/MAGICDATA-CLAM-CONVERSATION_CN) | 97,184 轮 × 15 主题 | 场景跃迁检测 | ★★☆ |
| [roleplay_multiturn_chat_1k_zh](https://huggingface.co/datasets/Minami-su/roleplay_multiturn_chat_1k_zh_v0.1) | ~1,000 对话 | 角色人格连续性 | ★☆☆ |

**关键问题**：这些数据集的标注维度和 RepEng 的 5 个控制维度不直接对齐。需要的是一个映射：
- 情绪标注（joy/sadness/anger...）→ emotion_valence + empathy
- 场景标注（formal/casual...）→ formality
- 创意vs分析任务 → creativity

这个映射可以通过 Phase 1 的投射方法自动推导——不需要手工对齐。

### 4.6 长期身份漂移

如果 Joi 和用户交互了几百轮，她的"河道"会不会变得越来越窄（收敛到固定模式）？还是保持探索性？

活系统需要某种"好奇心"或"随机扰动"来防止过早收敛。这类似于强化学习中的 exploration-exploitation 权衡。但这也可能只是过度设计 — 也许人类用户本身的多样性就足以维持探索。

## 5. 实验结果 (Phase 1–4)

### Phase 1: 语义投射方向验证 ✅

**核心问题**：`embed(对话) · V` 产生的漂移方向是否和人类直觉一致？

**方法**：用 Qwen3-8B 提取 12 段不同风格对话在 Layer 27 的 hidden state，投射到 5 个控制向量方向上，z-score 去均值后检查排名。

**结果**：

| Layer | Top-2 对齐率 |
|-------|-------------|
| 9 (浅层) | 58% |
| 18 (中层) | 67% |
| **27 (深层)** | **83%** |

具体对齐情况（Layer 27，mean-centered）：
- 悲伤对话 → **empathy 最强** ✓
- 技术问题 → **formality + confidence** ✓✓
- 头脑风暴 → **creativity + confidence** ✓✓
- 心理疏导 → **empathy 最强** ✓
- 闲聊/轻松 → **emotion 最强** ✓
- 正式报告 → **formality + confidence** ✓✓

**关键发现**：
1. 原始投射被模型基态偏移淹没 — **必须去均值**
2. 深层效果显著优于浅层
3. **框架验证通过** — 对话语义可以有效驱动漂移

### Phase 2: 漂移引擎原型 ✅

在一段 10 轮脚本对话上模拟了 `s(t+1) = clip(s(t) + η·u(t), Envelope)` 动力学。

**观察**：
- 闲聊阶段：emotion 和 empathy 轻微上浮
- 情绪深入阶段：empathy 持续攀升至峰值 +0.17
- 转向技术：formality 上浮，empathy 回落
- 创意发散：creativity 急剧上升（最强信号 +2.3σ）
- 回归轻松：多维度回调
- η=0.15 + momentum=0.7 产生平滑、符合直觉的轨迹
- 无包线碰撞（系数始终在安全区间内）

### Phase 3: 端到端闭环 ✅

**首次真正的 Joi 循环**：对话 → hidden state → 投射 → 漂移 → 系数注入 → 生成

5 轮闭环测试结果：
- 模型在每个语境下生成了上下文相关的、风格适应的回复
- 从闲聊到情绪支持到技术讨论到创意发散，风格自然过渡
- 平均 trigram_rep = 0.096（略高于保守阈值，因为 thinking 模式贡献了重复）
- **零包线碰撞** — 漂移始终在安全范围内
- 系数变化合理：formality 在技术讨论中升至 +0.32，empathy 在情绪话题中升高

### Phase 4: 包线体积 = 人格多变性度量 ✅

Monte Carlo 采样计算 14 个模型的 5D 安全包线体积：

| 排名 | 模型 | 保守包线占比 | 解读 |
|------|------|------------|------|
| 1 | **thinking-on** | **100%** | CoT 完全消除悬崖，全空间可用 |
| 2 | temp-1.5 | 95.4% | 高温度打开表达空间 |
| 3 | Qwen2.5-7B-Instruct | 94.2% | 对齐极大扩展安全区 |
| 4 | temp-1.0 | 85.5% | |
| 5 | Qwen3-14B-AWQ | 84.2% | 更大模型 = 更大包线 |
| ... | ... | ... | |
| 13 | **Qwen2.5-7B-Base** | **1.3%** | 未对齐，几乎全空间危险 |
| 14 | **lang-en (英语)** | **0.0%** | 英语输入极度放大悬崖 |

**关键发现**：
- **对齐是人格多变性的最强因子**（Base 1.3% → Instruct 94.2%，72× 提升）
- **CoT Thinking 赋予最大人格空间**（100% safe）
- **英语输入极度压缩包线**（0% safe）
- 这意味着：选 Joi 的底座模型时，对齐程度和推理能力比参数量更重要

## 6. 实验路线图（下一步）

### Phase 5: 自适应漂移率 + 场景跃迁检测

η 不应是常数。强信号（情绪爆发）应产生更大漂移，弱信号（闲聊）应低漂移。
场景跃迁（"好了聊正事"）应触发快速重定位。

### Phase 6: 模型自身输出的反馈回路

当前漂移只由用户输入驱动。模型输出是否也应产生漂移力？
需要评估自增强回路的风险（温暖→更温暖→过度温暖）。

### Phase 7: 长时间多轮验证

在 50+ 轮对话中验证：
- 轨迹是否收敛/发散？
- 人格一致性如何？
- 不同 η 下的"河道"形态差异

### Phase 8: 数据集扩展（HF/ModelScope）

利用外部对话数据集验证漂移轨迹在更多样化对话上的表现。
不是用于训练（漂移是投射，不是学习），而是用于压力测试。
